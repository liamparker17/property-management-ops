## Locked decisions for M3 (answered 2026-04-25)

These are binding for every M3 task. If a task description contradicts a locked decision, the decision wins.

1. **No approvals on maintenance.** Workflow is: request → assign contractor → work done → capture cost → cost debited from landlord's trust ledger, surfaces on landlord statement. No `MAINTENANCE_QUOTE_APPROVE` ApprovalKind, no approval routing for repairs. The existing `Approval` model remains used for **lease approvals only** (lease create/renew/rent change), which are out of scope for M3 to change further.

2. **Vendor schema keeps multi-quote support** (so a future WhatsApp Business cascade can store each contractor's response). M3 UI prioritises speed-to-dispatch over side-by-side quote comparison — the quote CRUD exists but the staff UI offers a single "assign contractor" flow that captures an optional first-quote value. WhatsApp cascade is deferred to M4+.

3. **Deposit settlement is immutable once finalised.** On lease termination: if no deductions, full `Lease.depositAmountCents` is released to tenant. If deductions exist, release `depositAmountCents - sum(charges)` to tenant. If `sum(charges) > depositAmountCents`, the shortfall is recorded as `DepositSettlement.balanceOwedCents` (positive integer); no auto-invoice generated — PM follows up manually. A new settlement cannot be opened once finalised.

4. **Inspection templates default to blank.** No per-unit-type starter checklists ship in M3. A future "save-as-template" feature is explicitly deferred.

5. **Inspection photos use direct Blob upload via signed URLs** (speed > server proxy). Client requests a signed upload URL from `/api/inspection-items/[id]/photos/signed-url`, PUTs directly to Blob, then POSTs metadata to `/api/inspection-items/[id]/photos` to register the `InspectionPhoto` row.

6. **Move-out charges are settlement-only** (`MoveOutCharge` rows on the `DepositSettlement`). No tenant invoice is generated for charges or for shortfalls. Shortfall is visible on the settlement statement.

7. **Either-signer SIGNED_OFF** — for MOVE_IN and MOVE_OUT inspections, either a tenant OR a staff/landlord signature is sufficient to flip `InspectionStatus` from COMPLETED to SIGNED_OFF. INTERIM inspections can be SIGNED_OFF by tenant only (no staff signature needed).

8. **Acceptance gate** mirrors M1/M2: typecheck + tests + nav validator + migrate status all green; `CODEBASE.md` refreshed.

## Environment variables added in M3

None. Vercel Blob credentials from M0/M1 are reused; signed upload URLs use the existing `BLOB_READ_WRITE_TOKEN`.

No ledger entry when `refundDueCents === 0`.
    - Stamps `finalizedAt`; flips `OffboardingCase.status` to `SETTLING`.
    - Writes audit entry.
  - `closeOffboardingCase(ctx, caseId): Promise<OffboardingCase>` — refuses if lease state ≠ `TERMINATED` or settlement not finalised; stamps `closedAt`; flips status to `CLOSED`.
